Preguntas

¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?

Es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor porque se hace saber a los usuarios del sistema de manera casi inmediata si ha habido un error en el llenado de datos, sin la necesidad de esperar la repuesta del servidor.

Además se podría decir que usar JavaScript para checar que sean válidos los inputs: Evitar errores de usuario, reduce la carga de procesamiento de los servidores, ya que comprueba todo antes de mandarlo al servidor, esto puede ayudar a reducir costos y mejorar la experiencia del usuario.

¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?

Hay varias formas de saltarse la seguridad de validaciones hechas con JavaScript, por ejemplo: La forma más sencilla de deshabilitar la validación para formularios con HTML5 es añadir el atributo novalidate al formulario, se puede hacer un script desde la consola para evitar la validación o se puede modificar la función directamente desde Java Script.

Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?

Es una buena practica porque hace una revisión de datos, los valida y se asegura que la información esté en el formato requerido por el servidor, de esta manera se ahorran recursos, se reducen gastos y tiempo.
