Conceptos

URL

URL (Uniform Resource Locator) es la dirección única y específica que se asigna a cada uno de los recursos en internet (sitios web, páginas web, textos, fotos, vídeos, etc.) para que puedan ser localizados por el navegador y ser visualizados por el usuario.

MIME

MIME (Multipurpose Internet Mail Extensions) es una serie de especificaciones dirigidas al intercambio de contenido a través de internet de todo tipo de archivos (texto, audio, vídeo, etc.) para identificar el formato de un archivo.

Métodos y Códigos de Errores HTTP

HTTP (Hypertext Transfer Protocol) es el protocolo de comunicación que permite las transferencias de información en la World Wide Web.

HTML5

HTML (HyperText Markup Language) es un lenguaje de marcado para la elaboración y definir el contenido de las páginas web. Básicamente se trata de un conjunto de etiquetas que sirven para definir el texto y otros elementos que compondrán una página web, como imágenes, listas, vídeos, etc. HTML5 es la quinta y última versión hasta ahora de html.

JS

JS (JavaScript) es un lenguaje de programación interpretado, permite crear mejoras en la interfaz de usuario y páginas web dinámicas.

PHP

PHP (Hypertext Preprocessor) es un lenguaje interpretado de código abierto, del lado del servidor, especialmente adecuado para el desarrollo web, ya que puede ser implementado en HTML para crear páginas web dinámicas. Las ventajas de PHP son su flexibilidad y su alta compatibilidad con otras bases de datos.

JSP

JSP (JavaServer Pages) es una tecnología que permite incluir código en Java en páginas web. El denominado contenedor JSP es el encargado de tomar la página, sustituir el código en Java que contiene el resultado de su ejecución, y enviarlo al cliente, de esta forma se pueden diseñar páginas con partes fijas y partes variables.

AJAX

AJAX (Asynchronous JavaScript And XML) es una técnica de desarrollo web para crear aplicaciones interactivas. Estas aplicaciones se ejecutan en el navegador de los usuarios mientras se mantiene la comunicación asíncrona con el servidor en segundo plano.

Preguntas

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es el medio y la World Wide Web es la red que corre actualmente sobre Internet. Internet es una red global de redes, mientras que la World Wide Web (www), es una recopilación de información a la que se accede a través de Internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET=El método GET solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos. Es utilizado únicamente para consultar información al servidor, muy parecidos a realizar un SELECT a la base de datos.

HEAD=El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta. Este método se utilizar para obtener información sobre un determinado recurso sin retornar el registro. Este método se utiliza a menudo para probar la validez de los enlaces de hipertexto, la accesibilidad y las modificaciones recientes.

POST=El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor. Es utilizado para solicitar la creación de un nuevo registro, es decir, algo que no existía previamente, es decir, es equivalente a realizar un INSERT en la base de datos.

PUT=El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. Se utiliza para actualizar por completo un registro existente, es decir, es parecido a realizar un UPDATE a la base de datos.

PATCH=El método PATCH es utilizado para aplicar modificaciones parciales a un recurso. Este método es similar al método PUT, pues permite actualizar un registro existente, sin embargo este se utiliza para actualizar solo un fragmento del registro y no su totalidad, es equivalente a realizar un UPDATE a la base de datos.

DELETE=El método DELETE borra un recurso en específico, o sea este método se utiliza para eliminar un registro existente, es similar a DELETE a la base de datos.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Cuando un servidor devuelve una respuesta HTTP con código 200 significa OK o que todo está bien y que no ocurrió ningún error y que la solicitud ha tenido exito.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Un error 404 significa que el servidor no puede o no procesará la petición debido a algo que es percibido como un error del clienteque, esto puede ser porque el usuario ingreso un recurso que no existe o un erro del desarrollador que no puso bien un link o el desarrollador lo puso a propósito por seguridad, por lo que no necesariamente es responsabilidad del desarrollador corregir el sitio web.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Un error 500 significa que ha habido un error en el servidor del sitio web, por lo que si es responsabilidad del desarrollador corregir el sitio web o la falla en el servidor.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Que un atributo HTML5 esté depreciado o desaprobado (deprecated) significa que la versión de html lo soporta pero que pronto se va a perder el soporte.
Ejemplos de elementos de HTML 4 que en HTML5 estén desaprobados: basefont, center, font, s, strike, u.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML 4 no fue diseñado fue creado, además no hace distinción entre contenido, estilo y comportamiento, todo está mezclado. No hay un diseño apegado a principios de ingeniería de software en capas. HTML5 fue diseñado, tiene una separación de capas y además es un conjunto de tecnologías que incluye CSS, Java Script y HTML; las incluye porque HTML es para el contenido y su semántica, CSS para la presentación y Java Script para el comportamiento.

¿Qué componentes de estructura y estilo tiene una tabla?

Las tablas tienen componentes de estructura y estilo: table (el contenedor principal), tr (representando a las filas contenedoras de las celdas) y td (representando a las celdas).

¿Cuáles son los principales controles de una forma HTML5?

Los principales controles de una forma HTML5 son: Texto, Selección, Botones y Selector de archivos.

Sobre el ciclo de vida y desarrollo de los sistemas de información: ¿Cuál es el ciclo de vida de los sistemas de información?¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de los sistemas de información es:

1. Identificación de Problemas, Oportunidades y Objetivos.
2. Determinación de los Requerimientos de Información.
3. Análisis de las Necesidades del Sistema.
4. Desarrollo y Documentación del Software.
5. Pruebas del Sistema.
6. Implantación y Evaluación del Sistema.
7. Funcionamiento y Mantenimiento.
8. Fin de la Vida Útil del Sistema.

El ciclo de desarrollo de sistemas de información es un circulo:

Analisis -> Diseño -> Desarrollo -> Pruebas -> Implementación -> Mantenimiento -> (Repetir)
